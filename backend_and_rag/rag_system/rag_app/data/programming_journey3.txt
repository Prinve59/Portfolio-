# Programming Journey (Part 3)

## Transition from Application Development to Engineering

After gaining confidence in backend development, Prince's focus gradually shifted from simply creating applications to understanding software engineering as a discipline.

Instead of asking only "How can I build this?", he increasingly asked questions such as:

- Why is this architecture chosen?
- Can this be optimized?
- What is the production-ready approach?
- How do large systems solve this problem?
- What are the trade-offs between different designs?

This shift represents an important transition from programming to engineering.

Programming involves writing code that works. Engineering involves designing systems that remain maintainable, scalable, reliable, and efficient over time.

Prince's later projects and discussions consistently reflect this engineering mindset.

---

# Data Structures and Algorithms

As project complexity increased, Prince recognized that writing code alone was not sufficient.

Efficient software requires efficient algorithms.

This realization led to deeper engagement with Data Structures and Algorithms (DSA).

Rather than treating DSA solely as interview preparation, Prince viewed it as a way to improve logical thinking and write better software.

Topics discussed and practiced include:

## Fundamental Structures

- Arrays
- Strings
- Linked Lists
- Stacks
- Queues
- Hash Maps
- Sets

These structures introduced concepts of data organization, memory usage, and efficient access patterns.

---

## Searching and Sorting

Prince explored standard searching and sorting techniques to understand how data can be processed efficiently.

Examples include:

- Linear Search
- Binary Search
- Bubble Sort
- Selection Sort
- Insertion Sort
- Merge Sort
- Quick Sort

The focus gradually shifted from implementation to understanding time complexity and choosing the right algorithm for different situations.

## Trees

Tree-based structures became an important part of Prince's algorithmic learning.

Topics include:

- Binary Trees
- Binary Search Trees
- Tree Traversals
- Lowest Common Ancestor
- Height and Depth

Tree problems helped strengthen recursive thinking and hierarchical reasoning.

## Graph Algorithms

As algorithmic knowledge expanded, graph problems became another important area.

Topics include:

- Graph Representation
- Depth-First Search (DFS)

Graph algorithms reinforced the importance of selecting suitable data structures for complex relationships.

## Advanced Data Structures

## Algorithmic Techniques

# Competitive Programming and Problem Solving

Prince's approach to programming problems emphasizes understanding over memorization.

Typical workflow:

1. Understand the problem statement.
2. Identify constraints.
3. Analyze edge cases.
4. Consider brute-force solutions.
5. Improve complexity.
6. Validate correctness.
7. Optimize further if necessary.

This structured reasoning is visible across many coding discussions.
